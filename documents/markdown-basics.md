# Markdown Basics

## 1. Headings

```md
# Title
## Section
### Subsection
```

## 2. Paragraphs
To create a new paragraph in Markdown, leave one empty line between lines of text.

```md
This is paragraph one.

This is paragraph two.
```

## 3. Line Breaks
There are several ways to force a line break.

### Option 1: Use bullet points

- `mkdir` creates a new folder
- `cd` moves into that folder

### Option 2: Add two spaces at the end of a line

`mkdir` creates a new folder  
`cd` moves into that folder

### Option 3: Use </br/>

`mkdir` creates a new folder<br>
`cd` moves into that folder

## 4. Bullet Lists

- item one
- item two
- item three

## 5. Numbered Lists

1. Step one
2. Step two
3. Step three

## 6. Inline Code
Backticks can highlight commands, filenames, and keywords.

Use `git status` to check the repository.
This is useful when writing about terminal commands.

## 7. Code Blocks
Code blocks are useful for commands and code examples.

Example:

```bash
git status
git remote -v
```
The word bash helps show that the content is a terminal command.

## 8. Bold Text

** can be used for bold text

Example:

**important**

## 9. Italic Text
* Italic text can be used for light emphasis.

Example:

*note*

## 10. Links
Markdown can create clickable links.

Example:

[GitHub](https://github.com)

## 11. Blockquotes
Blockquotes are useful for notes, reminders, or reflections.

Example:

> This is a note.

## 12. Horizontal Lines
Horizontal lines can separate sections clearly.

Example:

---

## 13. Tables
Tables are useful for organizing commands, meanings, or vocabulary.

Example:

| Command | Meaning |
|--------|---------|
| `ls` | list files |
| `cd` | change directory |

## 14. Checkboxes
Checkboxes are useful for tracking learning progress.

Example:

- [x] finished
- [ ] not finished

## 15. Images
Markdown can also display images.

Example:

![alt text](image-link)

This can be useful later in a README if I want to show screenshots.
